## 2025-04-14 - Missing ARIA Labels on Navigation Components
**Learning:** Found a recurring pattern of missing `aria-label` attributes on icon-only buttons (`<Button size="icon">`) and interactive elements across layout navigation components (`top-bar.tsx`, `sidebar.tsx`). This significantly degrades screen reader navigation for critical layout controls (mobile menus, search, sidebar expansion).
**Action:** When reviewing layout or navigation components, verify all icon-only buttons include descriptive `aria-label`s, and ensure stateful buttons (like expand/collapse) manage `aria-expanded` attributes appropriately.
## 2025-02-28 - Interactive Cards Missing Keyboard Access
**Learning:** Interactive UI cards (like `ItemCard` and `CollectionCard`) implemented using `div` or custom component containers without native button semantics were visually responsive to clicks but completely inaccessible via keyboard navigation (tabbing) or activation (Enter/Space). Furthermore, nested interactive elements (like the copy button or dropdown triggers) that relied entirely on group-hover for visibility became undiscoverable to keyboard-only users.
**Action:** When creating custom interactive container elements, always apply `role="button"`, `tabIndex={0}`, and handle keyboard activation (`onKeyDown` for Enter/Space). Additionally, any interactive sub-elements that are hidden by default and appear on hover (`group-hover:opacity-100`) must also be made visible on focus (`focus-visible:opacity-100`) to ensure they are discoverable by keyboard navigation.
