## 2024-04-24 - Missing Input Validation in Catch-All Routes
**Vulnerability:** The `/api/download/[...path]/route.ts` API route accepted an array of path segments directly from `params` without explicitly checking if any segments contained directory traversal payloads like `..`.
**Learning:** Next.js catch-all routes (`[...path]`) automatically parse the URL, but the segments themselves can still be crafted to traverse out of the intended directory when joined back into a string, potentially leading to unauthorized file access if concatenated naively with a base path.
**Prevention:** Always validate user-provided paths or URL segments, even when using modern routing abstractions. Ensure that sequences like `..`, `.`, and `/` are explicitly disallowed before relying on the path parameter to interact with files or external storage.
